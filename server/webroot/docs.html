REST API v1

API Reference

All /api/v1/ endpoints take JSON bodies and return JSON. Auth is Bearer token only.

Authentication

Every /api/v1/ request needs an Authorization: Bearer <token> header. No session cookie support.

Bearer token
curl -H "Authorization: Bearer tun_<your-token>" \
     http://localhost:1024/api/v1/status

Permission levels

read
  • GET /api/v1/status
  • GET /api/v1/agents
  • GET /api/v1/routes
  • GET /api/v1/agents/{id}
write
  • All read permissions
  • Port forwards CRUD
  • Reverse forwards CRUD
  • SOCKS proxies CRUD
  • Agent tag / ping / scan
admin
  • All write permissions
  • Bind connect control
  • CLI command execution
  • Server settings
  • Token management
Token management endpoints (/api/v1/tokens) require admin permission. Guard your admin tokens carefully they can create and revoke other tokens.

Error Responses

All errors return a JSON body with an error field and an appropriate HTTP status code.

Error body
{"error": "agent_not_found"}
StatusMeaning
401Bad or missing token
403Token does not have the required permission
404Resource not found
405Method not allowed
409Conflict, e.g. port in use
500Internal server error

Status

Health check. Returns agent count and active ports.

Example request
curl -H "Authorization: Bearer $TOKEN" \

  "agents":    3,
  "http_port": 1024,
  "tcp_port":  1080,
  "udp_port":  1081,
  "dns_port":  5300
}
200 OK

Routes

GET /api/v1/routes read List routes ›

Returns all routes with their agent and enabled/disabled state.

Example
curl -H "Authorization: Bearer $TOKEN" \
     http://localhost:1024/api/v1/routes
Response
[
  {"subnet": "192.168.1.0/24", "agent_id": "agent-1", "disabled": false},
  {"subnet": "10.10.10.0/24",  "agent_id": "agent-2", "disabled": true}
]
200 OK
PATCH /api/v1/routes write Enable / disable a route ›

Enable or disable proxying for a subnet. Disabled routes are skipped by the transparent proxy.

Request body

FieldTypeRequiredDescription
subnetstringyesCIDR subnet, must exist in routing table
disabledboolyestrue to disable, false to enable
Disable
curl -X PATCH -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"subnet":"192.168.1.0/24","disabled":true}' \
     http://localhost:1024/api/v1/routes
Enable
curl -X PATCH -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"subnet":"192.168.1.0/24","disabled":false}' \
     http://localhost:1024/api/v1/routes
Response
{"subnet": "192.168.1.0/24", "disabled": true}
200 OK 404 subnet_not_found

Port Forwards

Agent opens a port locally and forwards connections to the target.

GET /api/v1/forwards write List all forwards ›
Example
curl -H "Authorization: Bearer $TOKEN" \
     http://localhost:1024/api/v1/forwards
200 OK
POST /api/v1/forwards write Create forward ›

Request body

FieldTypeRequiredDescription
agent_idstringyesTarget agent ID
listen_portintyesPort to listen on the agent
target_hoststringyesDestination host
target_portintyesDestination port
protocolstringnotcp (default) or udp
Example
curl -X POST -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"agent_id":"agent-1","listen_port":8080,"target_host":"192.168.1.10","target_port":80,"protocol":"tcp"}' \
     http://localhost:1024/api/v1/forwards
201 Created 404 agent_not_found 409 port_already_in_use
DELETE /api/v1/forwards?id={listener_id} write Delete forward ›
Example
curl -X DELETE -H "Authorization: Bearer $TOKEN" \
     "http://localhost:1024/api/v1/forwards?id=<listener-id>"
200 OK 404 not_found

Bind Connect

Agent runs in bind mode (listens). Server dials out to it. Retries automatically until cancelled.

admin only. DELETE also kills the active agent session.
GET /api/v1/connect-bind admin List active bind connects ›
Example
curl -H "Authorization: Bearer $ADMIN_TOKEN" \
     http://localhost:1024/api/v1/connect-bind
Response
[{"addr": "172.16.1.5:9001", "active": true}]
200 OK
POST /api/v1/connect-bind admin Start bind connect loop ›
FieldTypeRequiredDescription
hoststringyesBind agent IP or hostname
portintyesBind agent listen port
Example
curl -X POST -H "Authorization: Bearer $ADMIN_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"host":"172.16.1.5","port":9001}' \
     http://localhost:1024/api/v1/connect-bind
201 Created 409 already_connecting
DELETE /api/v1/connect-bind?addr={host:port} admin Stop + disconnect ›

Kills the retry loop and drops the active connection if there is one.

Example
curl -X DELETE -H "Authorization: Bearer $ADMIN_TOKEN" \
     "http://localhost:1024/api/v1/connect-bind?addr=172.16.1.5:9001"
200 OK 404 not_found

Token Management

GET /api/v1/tokens admin List tokens ›

Lists tokens. The raw token string is only shown on creation.

Example
curl -H "Authorization: Bearer $ADMIN_TOKEN" \
     http://localhost:1024/api/v1/tokens
200 OK
POST /api/v1/tokens admin Create token ›
FieldTypeRequiredDescription
namestringyesToken label, max 64 chars
permissions[]stringnoread, write, or admin. Defaults to read.
Example
curl -X POST -H "Authorization: Bearer $ADMIN_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"name":"ci-read","permissions":["read"]}' \
     http://localhost:1024/api/v1/tokens
Response save the token now, it won't be shown again
{
  "id":          "a1b2c3d4",
  "token":       "tun_XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX=",
  "name":        "ci-read",
  "permissions": ["read"],
  "created_at":  "2026-04-02T10:00:00Z"
}
201 Created 400 name_required
DELETE /api/v1/tokens/revoke?id={token_id} admin Revoke token ›
Example
curl -X DELETE -H "Authorization: Bearer $ADMIN_TOKEN" \
     "http://localhost:1024/api/v1/tokens/revoke?id=a1b2c3d4"
200 OK 404 not_found

All actions below are POST /api/v1/agents/{id}/{action}.

POST /api/v1/agents/{id}/disconnect write Disconnect agent ›
Example
curl -X POST -H "Authorization: Bearer $TOKEN" \
     http://localhost:1024/api/v1/agents/agent-1/disconnect
200 OK
POST /api/v1/agents/{id}/reconnect write Request reconnect ›
Example
curl -X POST -H "Authorization: Bearer $TOKEN" \
     http://localhost:1024/api/v1/agents/agent-1/reconnect
200 OK
POST /api/v1/agents/{id}/tag write Set agent tag ›
FieldTypeRequiredDescription
tagstringyesMax 64 chars. Send empty string to clear.
Example
curl -X POST -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"tag":"dmz-pivot"}' \
     http://localhost:1024/api/v1/agents/agent-1/tag
200 OK
POST /api/v1/agents/{id}/ping write ICMP ping from agent ›
FieldTypeRequiredDescription
targetstringyesIP or hostname to ping
countintnoPing count (1–20, default 4)
Example
curl -X POST -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"target":"192.168.1.1","count":4}' \
     http://localhost:1024/api/v1/agents/agent-1/ping
200 OK
POST /api/v1/agents/{id}/discover write Discover live hosts on a subnet ›
FieldTypeRequiredDescription
subnetstringyesCIDR notation e.g. 192.168.1.0/24
timeout_msintnoPer-host timeout ms (default 300)
workersintnoConcurrent workers (default 100)
Example
curl -X POST -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"subnet":"192.168.1.0/24","timeout_ms":300,"workers":50}' \
     http://localhost:1024/api/v1/agents/agent-1/discover
200 OK
POST /api/v1/agents/{id}/portscan write Port scan from agent ›
FieldTypeRequiredDescription
targetstringyesTarget IP or hostname
portsstringnoPort range e.g. 1-1024 or 22,80,443 (default 1-1024)
protostringnotcp (default) or udp
Example
curl -X POST -H "Authorization: Bearer $TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"target":"10.10.10.5","ports":"1-1024","proto":"tcp"}' \
     http://localhost:1024/api/v1/agents/agent-1/portscan
200 OK
